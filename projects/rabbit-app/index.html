Project Overview

Rabbit Health Monitoring is an innovative Flutter application designed to solve a real-world problem: rabbit owners need a reliable way to track their pets' health. Rabbits are sensitive animals, and their health status can be inferred from the appearance and quantity of their feces. This app automates that analysis using cutting-edge machine learning.

Problem Statement

The Challenge

Rabbit owners and breeders struggle with:

  • Lack of objective health monitoring tools
  • Difficulty detecting early signs of illness
  • No standardized way to track feces observations
  • Limited access to veterinary consultation on routine observations
  • Manual record-keeping that is tedious and error-prone

Why This Matters

Rabbits are prey animals that hide illnesses well. By the time owners notice behavioral changes, the problem may be serious. Feces appearance is one of the earliest indicators of health issues. An automated, intelligent system can catch problems early and potentially save lives.

Key Features

📷 AI-Powered Analysis

Users can capture images of rabbit feces using the in-app camera. The YOLOv8 model automatically detects and classifies the feces, counts them, and determines health status. The system achieves this with custom-trained model weights and real-time inference.

🐰 Rabbit Profile Management

Create and manage multiple rabbit profiles with detailed information:

  • Name, breed, birthdate, and weight tracking
  • Custom profile photos from gallery or camera
  • Individual health history per rabbit
  • Quick search and filter across multiple rabbits

📊 Health Dashboard & History

Comprehensive health tracking with:

  • Color-coded health status (green = normal, orange = caution, red = critical)
  • Interactive charts showing feces trends over time
  • Detailed historical records with date/time timestamps
  • Quick health overview of all rabbits

💡 Smart Recommendations

The app provides intelligent, context-aware recommendations:

  • "Keep an eye on it" - for minor variations
  • "Consult your veterinarian" - for concerning trends
  • "Critical: See veterinarian immediately" - for serious issues

📱 Offline-First Architecture

All data is stored locally on the device:

  • No internet connection required for core functionality
  • Instant access to historical data
  • Privacy-first approach - all data stays on user's device

🔐 Reliable Data Persistence

SQLite database ensures:

  • ACID compliance for data integrity
  • Efficient queries for real-time health analysis
  • Scalable schema supporting future features

Technical Implementation Highlights

1. Custom Image Preprocessing Pipeline

The app implements a sophisticated image preprocessing pipeline for YOLOv8 inference:

// Letterbox image resizing to maintain aspect ratio // Converts various input sizes to 640×640 canvas // Adds black padding where needed // Result: Consistent input for YOLOv8 model

2. On-Device ML Inference

YOLOv8 runs entirely on the device with:

  • No cloud dependencies or API calls
  • Configurable confidence threshold (0.2) and IOU threshold (0.4)
  • Real-time performance optimized for mobile
  • Custom label filtering for relevant detections

3. BLoC State Management

Enterprise-grade architecture with clean separation of concerns:

  • HomeBloc manages rabbit list and filtering
  • RabbitControllerBloc handles individual rabbit operations
  • Event-driven architecture for scalability
  • Proper testing and state isolation

4. Database Architecture

Normalized SQLite schema with:

  • Version control (v2) supporting migrations
  • Foreign key constraints for referential integrity
  • Optimized queries for common operations
  • Support for future schema evolution

Challenges & Solutions

Challenge: Model Size & Performance

Problem: TensorFlow Lite models can be large and slow on mobile devices.

Solution: Used quantized YOLOv8 model with float32 precision for the optimal balance between accuracy and performance. Single-threaded inference for consistent, predictable behavior.

Challenge: Image Aspect Ratio Handling

Problem: YOLO expects square inputs, but photos are rectangular.

Solution: Implemented custom letterbox preprocessing that maintains aspect ratio while padding to 640×640, preserving all image information without distortion.

Challenge: Real-time Health Analysis

Problem: Determining health status from a single image could be inconsistent.

Solution: Algorithm combines feces count, type, and historical trends for context-aware recommendations that improve over time.

Challenge: Offline Data Sync

Problem: Managing data consistency when user captures many images without internet.

Solution: Local SQLite as source of truth with timestamped operations ready for future cloud sync when implemented.
